**Бизнес-процесс** — это повторяемая последовательность действий, в ходе которой ресурсы на входе преобразуются в конкретный результат на выходе. У каждого процесса есть границы: он начинается с конкретного события (триггера) и заканчивается достижением цели. Важнейшие элементы — это роли (участники), регламенты (правила) и материальные или информационные потоки.

У каждого процесса есть чёткие границы:
- **Начало (триггер):** событие, с которого всё запускается
- **Шаги:** последовательность действий
- **Участники:** люди или системы, которые выполняют действия
- **Правила:** условия, сроки, ограничения
- **Результат:** что получаем в конце

**Примеры бизнес-процессов из разных сфер:**

## 1. Ритейл (Ozon) — Возврат товара
| Элемент | Описание |
|--------|----------|
| **Вход** | Неподошедшая вещь + чек |
| **Действия** | Заявка в приложении → Одобрение возврата → Приёмка товара → Перевод денег |
| **Участники** | Покупатель, поддержка Ozon, склад, бухгалтерия |
| **Выход** | Деньги на карте покупателя |

## 2. Такси (Яндекс.Такси) — Поездка
| Элемент | Описание |
|--------|----------|
| **Вход** | Точка А и точка Б |
| **Действия** | Поиск водителя → Подача машины → Поездка → Оплата |
| **Участники** | Пассажир, водитель, алгоритмы Яндекса |
| **Выход** | Пассажир доставлен, деньги списаны |

## 3. Медицина — Запись к врачу
| Элемент | Описание |
|--------|----------|
| **Вход** | Желание пациента попасть на приём |
| **Действия** | Выбор специалиста → Выбор свободного времени → Подтверждение записи |
| **Участники** | Пациент, регистратура, врач |
| **Выход** | Подтверждённый талон на конкретное время |
